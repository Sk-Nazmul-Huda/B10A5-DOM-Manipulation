Important FAQ

What is the Document Object Model (DOM) ?

The DOM (Document Object Model) is a programming interface for web documents. It represents the structure of an HTML or XML document as a tree of objects, where each object corresponds to a part of the page (like elements, attributes, and text). The DOM allows programming languages (typically JavaScript) to interact with and manipulate the content, structure, and style of web pages dynamically.

How do you select an element from the DOM?

To select an element from the DOM, you can use methods like
1. getElementById()
2. getElementsByClassName()
3. querySelector()
4. querySelectorAll() in JavaScript.

What is event delegation in the context of the DOM, and why is it useful?

Event delegation is a technique in JavaScript where a single event listener is added to a parent element to handle events for its child elements. Instead of attaching individual event listeners to each child, the parent listens for events that bubble up from the children.

This is useful because it improves performance, especially when dealing with dynamically added elements, and reduces memory usage by minimizing the number of event listeners needed.

How do you manipulate an element's attribute and styles using the DOM?

To manipulate an element's attributes and styles using the DOM:
Attributes: Use setAttribute() to change an attribute, and getAttribute() to read it.

Code here:
element.setAttribute('src', 'image.jpg'); // Change attribute
let src = element.getAttribute('src'); // Get attribute
Styles: Use the style property to modify inline CSS styles.

Code here:
element.style.color = 'red'; // Change color
element.style.fontSize = '16px'; // Change font size
These methods allow you to dynamically change the content and appearance of elements on the page.

What is event bubbling in DOM?

Event bubbling is a concept in the DOM where an event, such as a click or key press, starts from the target element and then "bubbles" up to its parent elements, triggering any relevant event listeners along the way. This allows parent elements to handle events on their child elements without needing individual listeners on each child.

For example, if you click a button inside a div, the click event will first trigger on the button, then bubble up and trigger any click listeners on the parent div, and so on up the DOM tree.
